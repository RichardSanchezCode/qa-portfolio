## Project Overview

This project demonstrates manual API testing using **Postman** against the **JSONPlaceholder REST API**.

The objective of this project is to showcase essential API testing skills, including CRUD operations, request validation, response verification, test documentation, and API automation fundamentals.

The project follows QA best practices by including test cases, execution reports, evidence, traceability, and testing notes.

---

# Project Objectives

- Practice REST API testing.
- Learn API validation using Postman.
- Design professional Test Cases.
- Document Test Execution.
- Organize testing evidence.
- Apply QA documentation standards.
- Build a portfolio-ready QA project.

# Features Tested

The following API functionalities were tested:

- Retrieve User by ID (GET)
- Create User (POST)
- Update User (PUT)
- Delete User (DELETE)
- Query Parameters
- HTTP Headers
- Environment Variables
- Data Driven Testing (Configuration)
